# Plan 4: Real-time (ActionCable + Sidekiq)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement real-time video share notifications using ActionCable (WebSocket) and Sidekiq (background job).

**Architecture:** When a video is created, `VideoShareNotificationJob` is enqueued. Sidekiq picks it up and calls `ActionCable.server.broadcast("notifications", payload)`. All clients subscribed to `NotificationsChannel` receive the payload. ActionCable connection is authenticated via JWT passed as `?token=` query param.

**Tech Stack:** ActionCable, Sidekiq 7, Redis, JwtService (from Plan 2)

- [ ] **Step 3: Implement ActionCable connection with JWT auth**

`app/channels/application_cable/connection.rb`:
```ruby
module ApplicationCable
  class Connection < ActionCable::Connection::Base
    identified_by :current_user

    def connect
      self.current_user = find_verified_user
    end

    private

    def find_verified_user
      token = request.params[:token]
      return reject_unauthorized_connection unless token

      payload = JwtService.decode(token)
      return reject_unauthorized_connection unless payload
      return reject_unauthorized_connection if JwtDenylist.exists?(jti: payload['jti'])

      User.find_by(id: payload['user_id']) || reject_unauthorized_connection
    end
  end
end
```

- [ ] **Step 3: Implement VideoShareNotificationJob**

`app/jobs/video_share_notification_job.rb`:
```ruby
class VideoShareNotificationJob < ApplicationJob
  queue_as :default

  def perform(video_id)
    video = Video.includes(:user).find(video_id)
    ActionCable.server.broadcast(
      'notifications',
      {
        type: 'new_video',
        title: video.title,
        shared_by: video.user.email
      }
    )
  end
end
```
